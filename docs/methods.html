Baseline method

The Baseline method estimates daily dose from structured OMOP fields using a four-step cascade. It tries each method in order and uses the first that produces a non-NA result.

StepLabelFormulaRequires
M1originalPre-existing daily_dose or daily_dose_mg if numeric and > 0daily_dose or daily_dose_mg
M2tablets_freqtablets × freq_per_day × strength_mgtablets, freq_per_day, strength_mg — requires SIG parsing first
M3actual_duration(quantity × strength_mg) / (end_date − start_date + 1) — Burkard (2024) standard for oral tablets v0.1.7quantity, date columns, strength_mg
M4supply_based(quantity × strength_mg) / days_supply — fallback only v0.1.7quantity, days_supply > 0, strength_mg

Workflow

OMOP drug_exposure (all drugs, all routes)
          │
          ▼
  standardize_drug_name(drug_concept_name)  →  drug_name_std
          │
          ▼  filter_oral = TRUE
  ┌───────────────────────────────────┐
  │ Gate 1: oral route only           │
  │  classify_route(route_concept_name│
  │                 route_source_value)│
  │  keep: "oral" or NA (unknown)     │
  └───────────────────────────────────┘
          │
          ▼
  ┌───────────────────────────────────┐
  │ Gate 2: known systemic steroids   │
  │  drug_name_std ∈ pred_equiv_table │
  └───────────────────────────────────┘
          │
          ▼
  strength_mg
  ├─ amount_value  (amount_unit_concept_id == 8576, 0, or NA)
  └─ fallback: mg regex on drug_source_value
          │
          ▼
  Imputation cascade  (first non-NA wins)
  ├── M1  original       pre-existing daily_dose / daily_dose_mg
  ├── M2  tablets_freq   tablets × freq_per_day × strength_mg
  ├── M3  actual_dur     (quantity × strength_mg) / (end_date − start_date + 1)  ← Burkard formula
  └── M4  supply_based   (quantity × strength_mg) / days_supply                  ← fallback
          │
          ▼
  Plausibility cap: doses > 2000 mg/day → NA + warning

strength_mg derivation

Unit safety (v0.1.1 / v0.1.6) amount_value is accepted when amount_unit_concept_id is the OMOP mg concept (8576), is absent/unknown (NA), or is the OMOP "No matching concept" sentinel (0). Many production CDMs write 0 instead of NULL for an unmapped unit; this is treated identically to NA. If the unit is mcg (9655), g (8504), or any other explicit non-mg concept, the value is discarded and the function falls back to extracting mg from drug_source_value (e.g. "PREDNISONE 5 MG TABLET"). Before v0.1.6, concept 0 was incorrectly rejected as non-mg, causing all imputation methods to return "missing" at sites that use the 0 sentinel.

Plausibility cap

max_daily_dose_mg (v0.1.1) Records whose imputed dose exceeds max_daily_dose_mg (default 2000) are set to NA with a warning counting affected rows. Pass NULL to disable. Raise the threshold if your cohort includes high-dose pulse therapy (> 2000 mg/day IV methylprednisolone).

Example

baseline_df <- calc_daily_dose_baseline(
  drug_df,
  m2_sig_parse      = "auto",  # parse SIG to enable M2
  max_daily_dose_mg = 2000
)
table(baseline_df$imputation_method, useNA = "ifany")

NLP method

The NLP method parses the free-text sig column using a hierarchy of regex rules. Before parsing, it filters to oral systemic corticosteroids by route and drug name.

Parsing hierarchy

  1. Flag detection — free-text ("as directed"), taper language, PRN ("as needed")
  2. Tablet count — "2 tablets", "2 tabs", "2 caps", etc.
  3. Frequency — QD/daily/once daily (1×), BID/twice daily (2×), TID (3×), QID (4×), QOD (0.5×), q6h, q8h, q12h
  4. mg extraction (priority order):
    1. (X mg total) — explicit daily total in parentheses
    2. (X mg per dose) — per-administration total
    3. (X mg) — plain parenthetical; multiplied by tablet count if known
    4. X mg — bare mg anywhere in string
    5. v0.1.1 Strength fallback — when no mg in SIG: use amount_value (from drug_strength JOIN), then mg from drug_concept_name or drug_source_value
  5. Daily dose = mg_per_admin × freq_per_day
  6. Status: "free_text", "taper", "prn", "ok", "no_parse", "empty"
Strength fallback (v0.1.1) In live OMOP data the sig field typically contains administration instructions without a dose amount (e.g. "take 1 tablet daily"). The drug strength lives in drug_strength.amount_value, already fetched by the SQL query. The NLP pipeline uses this as a per-tablet strength fallback, promoting records from "no_parse" to "ok" when freq was parseable from the SIG.

Example

nlp_df <- calc_daily_dose_nlp(drug_df)
table(nlp_df$parsed_status, useNA = "ifany")

# Inspect "no_parse" records — which field is missing?
nlp_df |>
  dplyr::filter(parsed_status == "no_parse") |>
  dplyr::mutate(
    missing_field = dplyr::case_when(
      is.na(freq_per_day) & is.na(mg_per_admin) ~ "freq + mg",
      is.na(freq_per_day)                          ~ "freq only",
      is.na(mg_per_admin)                          ~ "mg only",
      TRUE                                          ~ "other"
    )
  ) |>
  dplyr::count(missing_field, sort = TRUE)

Advanced NLP method v0.1.2

calc_daily_dose_nlp_advanced() extends the standard NLP method with richer SIG vocabulary and taper schedule decomposition. It returns the same columns as calc_daily_dose_nlp() and is fully compatible with convert_pred_equiv() and build_episodes().

What is new

  • Word-form tablet counts — "one tablet", "two tabs", "half a tablet", "1/2 tab", "½ cap", compound forms like "one and a half tablets"
  • Weekly frequencies — "once weekly", "twice a week", "3 times a week", "q7d", "every 7 days" (→ 1/7 per day)
  • Monthly frequencies — "monthly", "once a month", "q30d" (→ 1/30 per day)
  • Every-N-days — "every 3 days", "q48h" (→ 0.5/day), "q72h" (→ 1/3 per day)
  • Sub-daily every-N-hours — "every 4 hours" / "q4h" (→ 6/day)
  • Taper schedule decomposition — when expand_tapers = TRUE, taper records are expanded into per-step rows

Taper schedule decomposition

Two decomposition strategies are tried in order:

  1. Explicit multi-step — SIG contains ≥ 2 blocks of the form "X mg [freq] for N [unit]" separated by "then", commas, or semicolons
  2. Decrement pattern — SIG states a starting dose and fixed reduction: "start 60 mg then decrease by 10 mg every week" → generates descending steps

Example: explicit multi-step taper

adv_df <- calc_daily_dose_nlp_advanced(drug_df, expand_tapers = TRUE)

# Compute actual step dates from prescription start
adv_df |>
  dplyr::filter(!is.na(taper_step)) |>
  dplyr::mutate(
    step_start_date = drug_exposure_start_date + step_start_day,
    step_end_date   = drug_exposure_start_date + step_end_day
  ) |>
  dplyr::select(person_id, drug_name_std, taper_step,
                step_start_date, step_end_date, daily_dose_mg, parsed_status)

Taper schedule parser v0.1.2

parse_taper_schedule(sig_text) # returns a tibble or NULL

Standalone function that decomposes a taper SIG string into a table of dose steps. Returns NULL when the SIG cannot be parsed into ≥ 2 steps. Never throws an error.

Output columnDescription
stepInteger step number (1, 2, …)
daily_dose_mgDaily dose for this step (mg/day)
freq_per_dayDoses per day (defaults to 1 when not stated)
duration_daysLength of this step in days
step_start_dayCumulative days from prescription start to step start (0-based)
step_end_dayCumulative days from prescription start to step end (inclusive)
parse_taper_schedule(
  "60 mg daily for 2 weeks, then 40 mg daily for 2 weeks, then 20 mg daily for 2 weeks"
)
#   step daily_dose_mg freq_per_day duration_days step_start_day step_end_day
#      1            60            1            14              0           13
#      2            40            1            14             14           27
#      3            20            1            14             28           41

parse_taper_schedule("start 60 mg then decrease by 10 mg every week")
#   step daily_dose_mg freq_per_day duration_days step_start_day step_end_day
#      1            60            1             7              0            6
#      2            50            1             7              7           13
#   ... (continues down to step 6: 10 mg)

SIG parsing details

parse_sig_one() — single record

parse_sig_one(sig_text) # returns a 1-row tibble

Never throws an error. Malformed or empty input returns parsed_status = "empty".

parse_sig() — vectorised

parse_sig(drug_df, sig_col = "sig")

Applies parse_sig_one() to every row and binds the results back onto drug_df.

parse_sig_one_advanced() / parse_sig_advanced()

Drop-in replacements for the standard parsers with the extended vocabulary (word-forms, weekly/monthly, every-N patterns). Return identical output columns.
