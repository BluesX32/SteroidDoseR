Data quality issues and how the package handles them

IssueHow it manifestsPackage response
Non-mg amount_value M3/M4 produces doses in billions (mcg treated as mg) v0.1.1 Unit concept ID checked; non-mg values discarded; string fallback used
amount_unit_concept_id = 0 All methods return "missing" — strength_mg = NA for every record v0.1.6 Concept 0 (OMOP "No matching concept" sentinel) now treated as unknown unit, identical to NA. Diagnostic warning emitted if strength_mg is still all-NA after both fallbacks.
SIG omits mg All NLP records fall to "no_parse" v0.1.1 amount_value used as per-tablet strength fallback
Large quantity × small days_supply supply_based (M4) dose explosion (e.g. 90-day supply with days_supply = 1) max_daily_dose_mg cap (default 2000 mg/day) NAs out outliers with a warning
Same-day start and end date actual_duration (M3) uses date range = 1 day → high dose Caught by max_daily_dose_mg cap
Missing sig column NLP method finds nothing to parse Empty sig column inserted automatically; parsed_status = "empty" for all; strength fallback still applies
Missing drug_exposure_end_date M4 and episode building fail Today's date substituted with a one-time warning; M1–M3 unaffected
Budesonide Equivalency factor is route-dependent (oral vs. inhaled differ by 10×) equiv_factor = NA for budesonide; pred_equiv_status = "na_factor"; requires manual review
Taper prescriptions A single record covers a decreasing-dose course parsed_status = "taper"; dose computation skipped (no stable daily dose). Use calc_daily_dose_nlp_advanced(expand_tapers = TRUE) to decompose.

Troubleshooting

All Baseline methods returning "missing"

  1. Check table(drug_df$amount_unit_concept_id, useNA = "ifany"). If the dominant value is 0, the fix in v0.1.6 resolves this — upgrade the package.
  2. Check table(baseline_df$imputation_method) — which method is producing the missing values?
  3. Look at drug_df$drug_source_value — does it contain "X mg" patterns that the string fallback can extract?

All NLP records are "no_parse" or "empty"

  1. Check whether sig is populated: table(is.na(drug_df$sig))
  2. If all NA, use sig_source = "drug_source_value" or rely on the strength fallback alone.
  3. If SIG exists but records still fail, inspect the failing patterns: nlp_df |> dplyr::filter(parsed_status == "no_parse") |> dplyr::count(sig, sort = TRUE) |> head(20)

Baseline doses are implausibly high

  1. Inspect amount_unit_concept_id: concept 8576 (mg), 0, and NA are accepted; all others are discarded.
  2. Check days_supply distribution: summary(drug_df$days_supply) — very small values inflate M3.
  3. The default max_daily_dose_mg = 2000 cap will NA out obvious outliers with a warning that counts affected rows.

No records after oral filter

Warning: "No oral corticosteroid records found after filtering"

Either route_concept_name is absent and/or drug names don't match the built-in steroid list. Use filter_oral = FALSE if your data is already pre-filtered to steroids, or check drug_df$route_concept_name and drug_df$drug_name_std.

Database connection fails

  1. Verify JDBC drivers: pkgbuild::has_build_tools(debug = TRUE)
  2. Test connection independently: DatabaseConnector::connect(connectionDetails)
  3. Check schema: the SQL queries reference @cdm_schema.drug_exposure — confirm your schema name and account permissions.
  4. Use create_connection_from_env(".env") to centralise all config and eliminate copy-paste errors.

Changelog

v0.2.5 Dashboard — Raw Records tab

  • feat(viz): launch_dose_dashboard(raw_list) — New optional raw_list parameter accepts a named list of record-level data frames (output of any calc_daily_dose_*() function, optionally passed through convert_pred_equiv()). Adds a Raw Records tab with rows colour-coded by method (matching the plot palette), per-column search filters, and a Download raw records (.csv) button. Backward-compatible: defaults to NULL; tab shows a placeholder message when omitted.
  • feat(viz): tabsetPanel layout — Main panel restructured into Timeline (dose trajectory plot), Episodes (episode-level summary table), and Raw Records (new record-level diagnostic table) tabs. All existing controls and download buttons are unchanged.

v0.2.0 Bug fix — injection filtering via drug_source_value

  • fix(utils-validate): classify_route() now checks drug_source_value as a third fallback — when both route_concept_name and route_source_value are absent or NA, the drug name string is now used to infer route. Many EHR systems encode route in the drug name (e.g. "METHYLPREDNISOLONE 125MG/2ML IV SOL"). All three imputation functions pass drug_source_value to the classifier so injections with no route columns are correctly excluded.
  • fix(utils-validate): classify_route() injection pattern now matches "intravenous" — the previous pattern intravein did not match the string "intravenous". Replaced with intraven which correctly matches "intravenous", "intravenously", etc. Added infusion, \\bsq\\b, and \\binjec\\b.
  • fix(utils-validate): oral pattern now matches \\btab\\b — abbreviated form "TAB" (common in drug name strings like "PREDNISONE 5MG ORAL TAB") is now recognised as oral route.

v0.1.9 Enhancement — new SIG patterns & tablet default

  • feat(nlp): "Once Oral", "Every evening Oral", "Nightly Oral" parse as freq_per_day = 1 — three common real-world SIG strings that previously returned NA frequency now resolve to once-daily. Leading whitespace is normalised before matching. Applies to both parse_sig_one() and parse_sig_one_advanced().
  • feat(nlp): tablets defaults to 1 when not specified in SIG — when a SIG contains a frequency and/or mg amount but no explicit tablet count (e.g. "5 mg daily", "Once Oral"), tablets is set to 1 rather than NA. This allows baseline M2 to complete for records where the prescriber omitted the tablet count.

v0.1.8 Bug fixes — defaults corrected

  • fix(baseline): filter_oral now defaults to TRUE — previously defaulted to FALSE, allowing injectables, inhalationals, and non-corticosteroids through unchecked. Now matches calc_daily_dose_nlp() and calc_daily_dose_nlp_advanced(). Pass filter_oral = FALSE if your input is already pre-filtered to oral steroids.
  • fix(baseline): m2_sig_parse now defaults to "auto" — previously defaulted to "warn", which silently skipped SIG-based M2 for all real-world OMOP data where tablets and freq_per_day columns are absent. SIG text is now parsed automatically when the sig column is present, enabling M2 before falling through to M3/M4.
  • fix(baseline/pipeline): run_pipeline(m2_sig_parse) default changed to "auto" — consistent with the above.
  • fix(baseline): cascade reorder — actual_duration (M3) before supply_based (M4) — Burkard (2024) formula (quantity × strength_mg) / (end_date − start_date + 1) is now the primary quantity-based method; days_supply-based computation is the fallback.

v0.1.6 Generalisation Bug fixes

  • fix(baseline): amount_unit_concept_id = 0 accepted as unknown — many production CDMs write concept 0 ("No matching concept") instead of NULL. Previously this caused strength_mg = NA for every record and returned "missing" for all methods. Concept 0 now treated identically to NA. A diagnostic warning is emitted when strength_mg is still all-NA after both fallbacks. (#BUG-8)
  • feat: drug_name_map parameter — standardize_drug_name(), convert_pred_equiv(), and all three imputation functions accept a user-supplied data frame (pattern, canonical_name) for site-specific synonyms or non-English brand names.
  • feat: equiv_table in imputation functions — all three imputation functions derive the allowed-drug list from the user-supplied equiv_table when filter_oral = TRUE, enabling studies beyond the built-in 7-drug table.
  • feat: max_daily_dose_mg in calc_daily_dose_nlp() — dose plausibility cap (default 2000 mg/day) added to the basic NLP function.
  • feat: SQL @vocab_schema — vocabulary table joins now use @vocab_schema instead of @cdm_schema, enabling sites with separate vocabulary schemas.
  • feat: dose_breaks / dose_labels in evaluate_against_gold() — dose-range stratification bins are now fully customisable.

v0.1.5 Baseline intermediate columns

  • feat(baseline): calc_daily_dose_baseline() now retains all four intermediate method columns — dose_from_original, dose_from_tablets_freq, dose_from_supply, dose_from_actual_duration — in its output, matching the Version2 Baseline.qmd column names.

v0.1.3 Visualisation

  • New: plot_patient_episodes() — static ggplot2 timeline overlaying all methods and gold standard as horizontal step segments, faceted by patient × drug.
  • New: launch_dose_dashboard() — interactive Shiny review dashboard with patient/drug filters, method toggles, line-width slider, and PDF/CSV download.
  • feat(episodes): build_episodes() now returns mean_daily_dose — duration-weighted mean daily dose within each episode.

v0.1.2 Advanced NLP

  • New: calc_daily_dose_nlp_advanced() — enhanced NLP pipeline with extended vocabulary, taper decomposition (expand_tapers), and dose plausibility cap.
  • New: parse_taper_schedule() — standalone taper SIG decomposer. Supports explicit multi-step and decrement patterns.
  • New: parse_sig_one_advanced() / parse_sig_advanced() — drop-in replacements for the standard parsers with richer vocabulary.

v0.1.1 Bug fixes

  • Baseline — unit-safe amount_value: non-mg amount_unit_concept_id values discarded before strength_mg calculation. Eliminates doses in the billions caused by mcg/g being treated as mg.
  • Baseline — plausibility cap: new max_daily_dose_mg parameter (default 2000). Implausible doses set to NA with a diagnostic warning.
  • Baseline — filter_oral: new parameter matching NLP method's oral filter. Default FALSE preserves backward compatibility.
  • NLP — strength fallback: when SIG contains no mg amount, amount_value (then drug_concept_name/drug_source_value) is used as per-tablet strength, promoting records from "no_parse" to "ok".
