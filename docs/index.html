SteroidDoseR

Automated corticosteroid dose extraction from OMOP CDM electronic health records

Version 0.1.6 R package JHU Myositis Research Group · OHDSI AgentDose 2025

Overview

SteroidDoseR computes prednisone-equivalent daily doses from OMOP CDM drug_exposure data and aggregates individual prescription records into continuous treatment episodes. It was built to support rheumatologic cohort studies — particularly myositis — where accurate steroid burden quantification is a primary outcome variable.

Two complementary imputation methods

MethodApproachBest when
Baseline Four-step cascade (M1–M4) using structured OMOP fields: pre-existing dose value, tablets×freq×strength, quantity/days_supply, quantity/actual_duration OMOP fields well-populated; SIG text absent or unreliable
NLP Rule-based regex parsing of free-text SIG prescription instructions; falls back to amount_value when SIG omits mg SIG column populated with structured abbreviations (QD, BID, TID, etc.)
Advanced NLP NLP extended with word-form tablet counts, weekly/monthly frequencies, every-N-days patterns, and taper schedule decomposition Taper SIGs that need per-step expansion

Analysis workflow

  1. Cohort selection — identify patients via ICD-10 codes, computable phenotype, or COHORT_PERSON_IDS = NULL for the entire database
  2. Fetch — query drug_exposure, concept, and drug_strength with a drug concept filter and patient filter
  3. Impute — compute daily_dose_mg via Baseline or NLP
  4. Convert — multiply by prednisone-equivalency factor → pred_equiv_mg
  5. Aggregate — gap-bridge consecutive prescriptions into episodes
  6. (Optional) Evaluate — compare episodes to a manually reviewed gold standard

Installation

Prerequisites

  • R ≥ 4.1
  • Core dependencies (auto-installed): dplyr, purrr, stringr, tibble, rlang, readr
  • For live OMOP CDM connectivity only: DatabaseConnector (≥ 3.0.0) and SqlRender (≥ 1.11.0) — not auto-installed
  • Windows source build: Rtools matching your R version

Install from local source

# Using devtools
devtools::install_local("path/to/SteroidDoseR/")

# Using pak (handles some Windows path issues better)
pak::local_install("path/to/SteroidDoseR/")

Database connectivity packages (optional)

install.packages("DatabaseConnector")
install.packages("SqlRender")

Verify installation

library(SteroidDoseR)
sessionInfo()  # should list SteroidDoseR under "other attached packages"

Quick start

Run the full pipeline against the bundled synthetic data — no database required:

library(SteroidDoseR)

# Load bundled 29-record synthetic dataset
extdata  <- system.file("extdata", package = "SteroidDoseR")
drug_exp <- readr::read_csv(file.path(extdata, "synthetic_drug_exposure.csv"),
                             show_col_types = FALSE)

# Wrap in a connector
con <- create_df_connector(drug_exp)

# One-call pipeline: fetch → impute → convert → episodes
episodes <- run_pipeline(con, method = "baseline")

episodes[, c("person_id", "drug_name_std", "episode_start",
             "episode_end", "n_days", "median_daily_dose")]

Switch to the NLP method with one argument change:

episodes_nlp <- run_pipeline(con, method = "nlp")
